# Enabling Encryption: HTTPS Setup for Open WebUI

This guide provides instructions for setting up **HTTPS encryption** for **Open WebUI**. While Nginx is used as an example, other reverse proxies like **Traefik** or **Caddy** can also be used. This setup assumes **Docker Compose deployment** for compatibility.

## Prerequisites

- **Docker** and **Docker Compose** are installed.
- Open WebUI is deployed and running.
- Basic knowledge of reverse proxy configuration.

<Tabs groupId="https-setup">

<TabItem value="nginx" label="Nginx Setup">

### Overview

In this example, **Nginx** is configured as a reverse proxy to enable HTTPS for Open WebUI. This setup uses **self-signed certificates** for demonstration purposes. You may substitute trusted certificates if desired.

---

### Nginx Configuration Files

1. **Create directories:**

   ```bash
   mkdir -p nginx/conf.d nginx/ssl
   ```

2. **Create the Nginx configuration files:**

**`nginx/conf.d/open-webui.conf`**:
```nginx
server {
    listen 443 ssl;
    server_name open-webui;

    ssl_certificate /etc/nginx/ssl/nginx.crt;
    ssl_certificate_key /etc/nginx/ssl/nginx.key;
    ssl_protocols TLSv1.2 TLSv1.3;

    location / {
        proxy_pass http://host.docker.internal:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

**`nginx/nginx.conf`**:
```nginx
user nginx;
worker_processes auto;
error_log /var/log/nginx/error.log warn;
pid /var/run/nginx.pid;

events {
    worker_connections 1024;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    log_format main "$remote_addr - $remote_user [$time_local] \"$request\" "
                    "$status $body_bytes_sent \"$http_referer\" "
                    "\"$http_user_agent\" \"$http_x_forwarded_for\"";
    access_log /var/log/nginx/access.log main;

    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;
    types_hash_max_size 2048;

    include /etc/nginx/conf.d/*.conf;
}
```

---

### Docker Compose Configuration

Use the following **Docker Compose** configuration to deploy Nginx with Open WebUI:

```yaml
version: '3.8'

services:
  nginx:
    image: nginx:alpine
    container_name: nginx_forwarding
    ports:
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
      - ./conf.d:/etc/nginx/conf.d
      - ./ssl:/etc/nginx/ssl
    restart: unless-stopped
```

### Using Self-Signed Certificates

The following configuration generates and uses **self-signed SSL certificates**.  
Note: **Self-signed certificates are not trusted by browsers by default.**

```yaml
version: '3.8'

services:
  nginx:
    image: nginx:latest
    container_name: nginx-proxy
    networks: ['demo']
    restart: unless-stopped
    ports:
      - 443:443
    volumes:
      - ./nginx/certs:/etc/nginx/certs
      - ./nginx/conf.d:/etc/nginx/conf.d
    depends_on:
      - nginx-init
    command: >
      /bin/sh -c "sleep 5 && nginx -g 'daemon off;'"
    environment:
      - NGINX_HOST=open-webui

  nginx-init:
    image: alpine
    container_name: nginx-init
    networks: ['demo']
    volumes:
      - ./nginx/certs:/etc/nginx/certs
    entrypoint: /bin/sh
    command:
      - "-c"
      - |
        apk add --no-cache openssl && \
        mkdir -p /etc/nginx/certs && \
        openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
        -keyout /etc/nginx/certs/selfsigned.key \
        -out /etc/nginx/certs/selfsigned.crt \
        -subj "/CN=open-webui"
```

> **Tip:** This setup can be extended to secure additional services by creating more configuration files in the `conf.d/` directory.
